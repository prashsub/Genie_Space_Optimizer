# Building the Genie Space Optimizer with APX

A step-by-step guide to building the Genie Space Optimizer as a Databricks App using [apx](https://github.com/databricks-solutions/apx) — the official Databricks Apps toolkit.

> **What is APX?** `apx` scaffolds a full-stack Databricks App with FastAPI + React + TypeScript + shadcn/ui, handles dev server, builds a deployable wheel, and integrates with `databricks bundle deploy`. It replaces manual Vite/React/FastAPI setup with a unified, AI-friendly workflow.

---

## Prerequisites

| Tool | Version | Install |
|------|---------|---------|
| `apx` | ≥0.3.5 | `curl -fsSL https://databricks-solutions.github.io/apx/install.sh \| sh` |
| Databricks CLI | ≥0.230 | `curl -fsSL https://raw.githubusercontent.com/databricks/setup-cli/main/install.sh \| sh` |
| Databricks profile | configured | `databricks configure --profile dev` |
| Python | ≥3.11 | APX auto-installs `uv` for Python management |
| Node/Bun | — | APX auto-installs `bun` for frontend builds |

**Databricks workspace requirements:**
- Unity Catalog enabled with a catalog and gold schema
- At least one Genie Space to optimize
- SQL Warehouse accessible to the app's service principal
- Permissions to create Databricks Jobs and Apps

## Step 7: Create the Optimization Harness

**Reference:** `docs/genie_space_optimizer/02-optimization-harness.md`

### `optimization/harness.py`

A library of stage functions powering the 6-task Databricks Job, plus a convenience `optimize_genie_space()` for testing. This is a deterministic 6-stage pipeline:

```
PREFLIGHT → BASELINE EVAL → ENRICHMENT → LEVER LOOP (1-6) → FINALIZE → DEPLOY (optional)
```

**Key functions:**
- `optimize_genie_space(space_id, catalog, schema, domain, ...)` → OptimizationResult
- `_run_preflight(...)` — fetch config, UC metadata, validate benchmarks, create MLflow experiment
- `_run_baseline(...)` — run 8-judge evaluation, establish baseline scores
- `_run_lever_loop(..., apply_mode="genie_config")` — for each lever (1-6): analyze failures → propose → apply → evaluate → accept/rollback. `apply_mode` controls where patches are written.
- `_run_finalize(...)` — repeatability test, promote best model, generate report
- `_safe_stage(...)` — error handler that writes failures to Delta

Also create supporting modules:
- `optimization/preflight.py` — preflight logic
- `optimization/models.py` — MLflow LoggedModel management
- `optimization/report.py` — report generation

**Source:** Refactor from `orchestrator.py`. Key changes:
- Replace JSON file I/O → Delta writes via `state.py`
- Replace `subprocess.run(["databricks", ...])` → `WorkspaceClient().jobs.submit_run()`
- Replace CLI args → function parameters
- Replace `sys.path` hacks → direct imports

### 8a. Multi-Task Job — 5 Notebook Entry Points

The optimization runs as a **single Databricks Job with 5 tasks**. Each task is a separate notebook calling one stage function from `harness.py`. Tasks pass data via `dbutils.jobs.taskValues`.

| Notebook | Task Key | Stage Function | Key Task Values Out |
|----------|----------|---------------|---------------------|
| `jobs/run_preflight.py` | preflight | `_run_preflight()` | run_id, model_id, experiment_name |
| `jobs/run_baseline.py` | baseline_eval | `_run_baseline()` | baseline_scores, thresholds_met |
| `jobs/run_lever_loop.py` | lever_loop | `_run_lever_loop()` | final_scores, best_model_id |
| `jobs/run_finalize.py` | finalize | `_run_finalize()` | final_status, report_path |
| `jobs/run_deploy.py` | deploy | `_run_deploy()` | held_out_accuracy |

Each notebook follows the pattern: read widgets + upstream task values → call stage function → set task values for downstream.

See `06-jobs-and-deployment.md` Section 1 for complete notebook source code.

### 8b. `jobs/run_evaluation_only.py` — Standalone evaluation notebook

Same pattern but calls `run_evaluation()` from `optimization/evaluation.py`.

### 8c. Job YAML — `resources/genie_optimization_job.yml`

Defines the multi-task job with `depends_on`, `condition_task` (run_if), and per-task timeouts/retries. See `06-jobs-and-deployment.md` Section 1 for full YAML.

   ```typescript
       data?.status && ['COMPLETED','FAILED','CANCELLED'].includes(data.status)

   });
   ```

2. **Progress bar:** `completedSteps / 5` with gradient (red → green)

5. **Time estimation:** For Steps 3 and 5, estimate remaining time as `(benchmarkCount × 12s) - elapsed`.

## Step 13: Wire Up the Optimization Flow End-to-End

With everything deployed, the full flow is:

```
User opens app → Dashboard (GET /api/genie/spaces)
  → Clicks SpaceCard → Space Detail (GET /api/genie/spaces/:id)
    → Clicks "Start Optimization" → POST /api/genie/spaces/:id/optimize
      → Backend creates QUEUED row in Delta + submits Databricks Job
      → Frontend navigates to Pipeline View (GET /api/genie/runs/:runId, polls 5s)
        → Multi-task job runs 5 tasks:
            preflight → baseline_eval → lever_loop → finalize → deploy
            (each task writes to Delta; lever_loop writes lever-level detail)
        → Frontend shows real-time step progress, lever sub-progress, scores
      → Pipeline completes → "View Results" button
        → Comparison View (GET /api/genie/runs/:runId/comparison)
          → User reviews diff → Apply or Discard
            → POST /api/genie/runs/:runId/apply (or /discard)
            → Navigate back to Dashboard
```

## Key APX Patterns to Follow

| Pattern | APX Way | NOT This |
|---------|---------|----------|
| Auth (app-level) | `Dependencies.Client` | `WorkspaceClient()` global |
| Auth (user-level) | `Dependencies.UserClient` | Manual token handling |
| Config | `Dependencies.Config` (Pydantic) | `os.environ.get()` |
| Router | `create_router()` | `APIRouter()` |
| API prefix | `api-prefix` in `pyproject.toml` | Hardcoded `/api/genie` in routes |
| Frontend API | Auto-generated hooks from OpenAPI | Manual `fetch()` calls |
| UI components | `apx components add <name>` (shadcn/ui) | Custom CSS from scratch |
| Routing | TanStack Router (file-based) | React Router v6 |
| Package manager | `uv` (Python), `bun` (JS) | pip, npm |
| Build | `apx build` → wheel | Manual Vite build |
| Deploy | `databricks bundle deploy` | Manual uploads |
| Dev server | `apx dev start` (all-in-one) | Separate uvicorn + vite |
