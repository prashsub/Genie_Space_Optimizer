Pattern 4

Tracing — Business-Context Observability

@mlflow.trace captures every predict call. start_span traces multi-step LLM reasoning as a span tree. The key insight: tag traces with business context (question_id, iteration, lever) — not just technical metadata.

@mlflow.trace
def genie_predict_fn(question, **kwargs):
    mlflow.update_current_trace(tags={
        "question_id": kwargs["question_id"],
        "genie.iteration": str(iteration),
        "genie.lever": str(lever),
    })
    result = run_genie_query(w, space_id, question)
    return {"response": result["sql"]}
start_span — LLM reasoning
with mlflow.start_span("strategy") as s:
    s.set_inputs({"clusters": len(c)})
    with mlflow.start_span("triage") as t:
        triage = call_llm_for_triage(c)
        t.set_outputs({"AGs": len(triage)})
    for ag in triage:
        with mlflow.start_span(f"detail_{ag}"):
            call_llm_for_details(ag)
Principle: @mlflow.trace is the single highest-ROI investment in a GenAI app. Tag with business context. Use start_span for multi-step LLM chains.

Takeaways

Design Principles for Production GenAI

1
Trace first, build second
Add @mlflow.trace on your predict function before anything else. Observability compounds.
2
Version prompts immediately
Even a rough draft. Version 1 in a registry beats a string buried in source code.
3
Create eval data early
Even 5 benchmarks. You cannot improve what you cannot measure.
4
Feedback over numbers
Structured Feedback with metadata is the difference between "broke" and "here's the fix."
5
LoggedModel everything
Config, prompts, system state — version it all. Rollback is when, not if.

"MLflow isn't a model training tool that happens to support GenAI. It's a GenAI lifecycle platform — experiments, prompts, datasets, evaluation, tracing, versioning, human review — that happens to also support model training. Treat it as the latter, and everything clicks."
